# Decision Records

Create a decision record only when a choice constrains future work, changes a public contract, or would otherwise be repeatedly reconsidered. Use `ADR-XXXX-short-name.md`; never rewrite an accepted decision silently—supersede it with a new record.

## Minimal template

```md
# ADR-XXXX: Decision title

- Status: proposed | accepted | superseded
- Date: YYYY-MM-DD
- Work package: WP-...

## Context
What verified problem or constraint required a decision?

## Decision
What is the chosen rule or architecture?

## Consequences
What becomes easier, harder, required, or intentionally unsupported?

## Evidence
Links to code, tests, issues, benchmarks, or runtime reproduction.
```
